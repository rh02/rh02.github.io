本系列主要是ELK的实操笔记部分，主要分享一下近期关于ELK的知识，包括它的运行原理、工作方式、基本特性和 Lucence的相关知识。希望以博客的形式能够保存下来，供后期查阅。

本序列借以希望能够提供该应用支持云上多主机多用户部署，以及kubernetes部署解决方案。

什么是ELK？

ELK 是三个开源软件的缩写，分别是：Elasticsearch，Logstash 和 Kibana。Elasticsearch 是一个搜索和分析引擎，底层的搜索引擎是 Lucence。 Logstash 是一个日志数据处理系统，可以从多数据源获取数据，并将数据转发给 Elasticsearch 的存储系统。 Kibana是一个数据可视化软件，可以让用户对Elasticsearch中数据进行图表的可视化展示。

ELK 经常被用于作为日志数据的存储系统，工作方式为将主机中的日志通过文件或者系统日志（syslog）的方式传递给 Logstach 进行解析（简单的讲，正则表达式）， 然后在存入 ElasticSearch 进行查询和统计，最后经过 Kibana 将统计的结果进行可视化展示。工作流程图如下图：

在学习ELK组件之前，需要了解以下名词和专业术语：

名词/术语 解释
index Elasticsearch的index相当于一个数据库。
type 数据库中的一张表。
id 唯一，类似于 Primary Key。
node 节点是Elasticsearch实例(Instance)，可以看成集群(Cluster)中的一個节点。一台机器可以运行多个实例，但是同一台机器上的实例在配置上要确保http和tcp Port不同。
cluster 表示一个集群，集群中有多个节点，其中有一个节点为 master， 该主节点类似于分布式中的大多数主节点一样是经过选举产生，主节点是针对该集群内部来讲的。
document 表示文档，代表了能够被索引的最小数据单元。在一个Index或者Type中，可以存储多个文档。虽然文档是存储在Index中的，但是实际上，它需要被索引或者分配到Index上的一个Type上。
shards 代表索引的分片，Elasticsearch 可以把一个完整的索引分成多个粉片，有点事能够讲一个大的索引拆分到多个，并且分布到多个节点上，对于分布式系统是非常有用的，可以实现分布式搜索，加快处理速度。分片的数量只能在创建前指定，如果索引创建后不能更改。
replicas 代表索引副本，Elasticsearch可以设置多个索引副本，副本的作用一是提高系统的容错性，当某个节点的分片损坏或者缺水，这时可以从副本中进行恢复。二是可以提高 ElasticSearch的查询效率，Elasticsearch会自动的对搜索请求进行负载均衡。

实操部分主要针对kubernetes安装与部署（后期更新），和容器化安装与部署。容器安装又分成单个组件安装和docker-compose部署。

单个安装

安装 ElasticSearch

从官方镜像源拉取 ElasticSearch 镜像。

docker pull docker.elastic.co/elasticsearch/elasticsearch:6.2.4

启动 ElasticSearch。

docker run -p 9200:9200 -p 9300:9300 --name elasticsearch -e "discovery.type=single-node" \
  docker.elastic.co/elasticsearch/elasticsearch:6.2.4

具体启动参数以及配置详见： https://hub.docker.com/_/elasticsearch/

在elasticsearch的docker版本文档中，官方提到了vm.max_map_count的值在生产环境最少要设置成262144。设置的方式有两种

  1. 永久性的修改,在/etc/sysctl.conf文件中添加一行：

    vm.max_map_count=262144

  2. 临时修改，进入正在运行的机器。

    sysctl -w vm.max_map_count=262144

安装 kibana

docker pull docker.elastic.co/kibana/kibana:6.2.4
docker run -d -p 5601:5601 --link elasticsearch -e \
  ELASTICSEARCH_URL=http://elasticsearch:9200 docker.elastic.co/kibana/kibana:6.2.4

安装 Logstash

logstash我们只让它进行日志处理，处理完之后将其输出到elasticsearch。

docker pull docker.elastic.co/logstash/logstash:6.2.4
docker run --rm -it --name logstash --link elasticsearch -d  \
  docker.elastic.co/logstash/logstash:6.2.4

安装 ELK Docker

docker pull sebp/elk

启动ELK

docker run -p 5601:5601 -p 9200:9200 -p 5044:5044 -it --name elk sebp/elk

第一次启动时，出现了 Starting Elasticsearch Server [fail]的错误，解决方案：内存小造成的，另一个原因就是沒有配置 vm.max_map_count 选项，经过上面修改就可以正常启动了。

检查 ElasticSearch 能否正常启动，可以在浏览器上输入：

http://your_ip:9200

初始用户名和密码是 elastic / password

检查 Kibana 能否正常启动，可以在浏览器上输入：

http://your_ip:5601

如果出现下面的情景，就证明启动成功。

使用 docker-compose 安装

docker-compose.yml

version: '2'
services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:6.2.4
    ports:
      - "9200:9200"
      - "9300:9300"
    environment:
      ES_JAVA_OPTS: "-Xms1g -Xmx1g"
    networks:
      - elk
  logstash:
    image: docker.elastic.co/logstash/logstash:6.2.4
    ports:
      - "5000:5000"
    networks:
      - elk
    depends_on:
      - elasticsearch
  kibana:
    image: docker.elastic.co/kibana/kibana:6.2.4
    ports:
      - "5601:5601"
    networks:
      - elk
    depends_on:
      - elasticsearch
networks:
  elk:
    driver: bridge


如果使用 elk 套装安裝，docker-compose.yml 內容如下：

elk:
  image: sebp/elk
  ports:
    - "5601:5601"
    - "9200:9200"
    - "5044:5044"

启动或重新启动定义在docker-compose.yml 的所有软件，可以加上 -d，改成在后台启动。

$ docker-compose up -d
