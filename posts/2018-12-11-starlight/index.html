翻译自 https://npf.io/2018/12/starlight. 翻译水平能力有限，还望包涵。本次主要讲解 starlight.

Starlight 主要封装了 google 的 Go语言实现的 Starlark, Startlark 是一种类 python 语言的语言（可以看成 python 语言的子集），最初是出现在Bazel构建工具中。 Starlight通过编写简单的类 python 脚本（在 Bazel中是 .zel， 在 Starlight 是 .star），它可以让用户能够非常容易地扩展当前的应用，这些脚本与当前的Go代码无缝地交互，而没有模板。

什么是 Starlight

Starlark 是 python 语言的一个子集，它删除了python语言中一些更高级的特性，但保持了易于读写的功能。出于本文的目的，为了避免 starlight（go pkg）和 starlark（语言）之间的混淆，我将代码称为 python（因为starlark 代码是 python 代码的子集），但是有一些小的区别（在前面的链接中描述）。

解释器

解析器和运行程序 由谷歌的 BAZEL 团队维护，Go编写的Starlark-go 。Starlight 封装了Starlark-go，这使得它更容易使用Starlark-go。

为什么不直接使用starlark-go？

Starlark-go API的问题在于，它更适合作为配置使用，因此它假设用户想从 starlark 获取信息并传递给 Go 程序。然而很难把 Go程序的信息（变量等） 传递给 Starlark 脚本。

轻松的数据双向传递

Starlight 的 适配器 使用反射机制来自动使任何 Go 语言中的值在 starlark 脚本中可用。可以将 *http.Request 传递到 Starlight 脚本吗？ 当然可以！ 并且你可以在 Python 中使用 name = r.URL.Query()["name"][0] 来获得 Query 变量 name的值。

100% 安全

starlark 和 starlight 的优点在于脚本可以 100％ 的安全运行。默认情况下，他们没有权限访问项目或系统的其他部分 – 它们无法写入磁盘或连接到Internet。 他们对外界的唯一访问权限就是用户给予它们的。因此，运行不受信任的脚本是安全的（只要你没有给它们运行危险的函数，比如 os.RemoveAll()）。但与此同时，如果你只运行受信任的脚本，可以随意使用它们（http.Get）

例子

下面是一个 Web 服务器的示例，它根据运行的 python 脚本更改其输出。这是完整的代码，它不会因为可读性而被截断……这就是它所需要的。

首先是 go web 服务器代码。非常标准的写法，除了调用 starlight API 的几行代码外……

package main
import (
    "fmt"
    "log"
    "net/http"

    "github.com/starlight-go/starlight"
)

func main() {
    http.HandleFunc("/", handle)
    port := ":8080"
    fmt.Printf("running web server on http://localhost%v?name=starlight&repeat=3\n", port)
    if err := http.ListenAndServe(port, nil); err != nil {
        log.Fatal(err)
    }
}

func handle(w http.ResponseWriter, r *http.Request) {
    fmt.Println("handling request", r.URL)
    // here we define the global variables and functions we're making available
    // to the script.  These will define how the script can interact with our Go
    // code and the outside world.
    globals := map[string]interface{}{
        "r":       r,
        "w":       w,
        "Fprintf": fmt.Fprintf,
    }
    _, err := starlight.Eval("handle.star", globals, nil)
    if err != nil {
        fmt.Println(err)
    }
}

下面是python代码 handle.star:

# Globals are:
# w: the http.ResponseWriter for the request
# r: the *http.Request
# Fprintf: fmt.Fprintf
# for loops and if statements need to be in functions in starlark
def main():
  # Query returns a map[string][]string
  
  # this gets a value from a map, with a default if it doesn't exist
  # and then takes the first value in the list.
  repeat = r.URL.Query().get("repeat", ["1"])[0]
  name = r.URL.Query().get("name", ["starlight"])[0]
  for x in range(int(repeat)):
    Fprintf(w, "hello %s\n", name)
  # we can use pythonic truthy statements on the slices returned from the map to
  # check if they're empty.
  if not r.URL.Query().get("repeat") and not r.URL.Query().get("repeat"):
    w.Write("\nadd ?repeat=&name= to the URL to customize this output\n")
  w.Write("\ntry modifying the contents of output.star and see what happens.\n")
main()


可以通过运行 go get github.com/starlight-go/starlight 并在 example 文件夹中使用 go run main.go 来运行此示例。然后可以更新 python 并在下次访问服务器时观察是否有变化。主要使用starlight.Eval 该方法，它每次访问时都要重新读取和编辑脚本。

比如修改 Python 片段代码为

for x in range(int(repeat)):
  Fprintf(w, "hello %s\n", name)
  Fprintf(w, "repeat %d\n", x)

缓存

在生产环境中，可能只想读取一次脚本并解析一次。你可以用 starlight 的 Cache 机制。该缓存采用一系列目录来查找脚本，它将按需读取和解析，然后将解析的对象存储在内存中供以后使用。它还可以使用 load() 方法调用使用缓存进而加载它们依赖的对象。
